# Settings Configuration System Plan

## Problem

Current settings implementation is redundant and error-prone:

- Each setting requires manual handling in server load, form action, and UI
- Adding new settings touches multiple files with repetitive boilerplate
- UI section has copy-paste bugs (wrong variable bindings)
- `darkmode` is extracted in server but never saved to DB

## Solution

Create a centralized settings configuration that drives both server and UI automatically.

### Step 1: Create Settings Config Schema

Create `/src/lib/config/settings.ts` that defines:

```ts
export type SettingType = 'boolean' | 'string' | 'select';

export interface SettingDefinition {
	key: string; // Database column name
	label: string; // Display label
	type: SettingType; // Input type
	default: boolean | string; // Default value
	category: string; // UI section grouping
	icon: string; // Lucide icon component name
	options?: readonly string[]; // For 'select' type only
}
```

### Step 3: Create Settings Service (`/src/lib/server/settings.ts`)

```ts
import { settingsConfig } from '$lib/config/settings';
import { db } from '$lib/server/db';
import { userSettings } from '$lib/server/db/app.schema';
import { eq } from 'drizzle-orm';

export function getDefaults() {
	return Object.fromEntries(settingsConfig.map((s) => [s.key, s.default]));
}

export async function loadUserSettings(userId: string) {
	const existing = await db.query.userSettings.findFirst({
		where: eq(userSettings.userId, userId)
	});
	return existing ?? { userId, ...getDefaults() };
}

export async function updateUserSettings(userId: string, formData: FormData) {
	const values: Record<string, boolean | string> = { userId };

	for (const setting of settingsConfig) {
		if (setting.type === 'boolean') {
			values[setting.key] = formData.get(setting.key) === 'true';
		} else {
			values[setting.key] = (formData.get(setting.key) as string) ?? setting.default;
		}
	}

	await db.insert(userSettings).values(values).onConflictDoUpdate({
		target: userSettings.userId,
		set: values
	});
}
```

### Step 4: Refactor `+page.server.ts`

Simplifies to ~15 lines:

```ts
import { loadUserSettings, updateUserSettings, getDefaults } from '$lib/server/settings';
import { settingsConfig } from '$lib/config/settings';

export const load: PageServerLoad = async (event) => {
	// ... auth check ...
	return { settings: await loadUserSettings(userId), config: settingsConfig };
};

export const actions: Actions = {
	updateSettings: async (event) => {
		// ... auth check ...
		await updateUserSettings(userId, await event.request.formData());
		return { success: true };
	}
};
```

### Step 6: Refactor `+page.svelte`

```svelte
<script lang="ts">
  import SettingsRow from '$lib/components/settings/SettingsRow.svelte';

  let { data } = $props();

  // Group settings by category
  const grouped = $derived(
    Object.groupBy(data.config, s => s.category)
  );
</script>

{#each Object.entries(grouped) as [category, settings]}
  <section>
    <p class="section-label">{category}</p>
    <div class="settings-card">
      {#each settings ?? [] as setting}
        <SettingsRow
          {setting}
          value={data.settings[setting.key]}
          onchange={(val) => /* submit */ }
        />
      {/each}
    </div>
  </section>
{/each}
```

### Step 7: Update Database Schema

Add `darkmode` boolean column to `userSettings` table (migration required).

## Benefits

1. **Single source of truth** - Adding a setting = one entry in config array
2. **No copy-paste errors** - Server and UI generated from same config
3. **Type-safe** - TypeScript ensures consistency
4. **Testable** - Config can be unit tested independently
5. **Consistent UI** - All settings render with same structure
